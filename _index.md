## About

colvmn (pronounced "column" — the `v` is Latin-styled for `u`) is a tiny layout engine for static sites. Pages are described by `_index.md` with YAML frontmatter, or `_index.json`. The engine renders them to static HTML at build time and to live DOM in the browser, so content is visible to JavaScript-free crawlers and to humans with JS enabled.

This page is itself generated by colvmn — the engine is self-hosting.

## How it works

A page is a directory containing:

- `index.html` — a thin shell that loads `style.css` and `layout/layout.js`
- `_index.md` or `_index.json` — the page's content, as frontmatter plus markdown or as structured JSON

At build time, `static-gen.js` walks the site tree, finds every page directory, runs the layout engine against each one, and writes the rendered HTML back into `index.html`. At runtime, `layout.js` does the same thing in the browser, replacing the pre-rendered content with a freshly rendered copy. The two paths produce identical output.

Content blocks are small classes in `layout/` — `ContentText`, `ContentCards`, `ContentTable`, `ContentTimeline`, and so on. Each reads a slice of the page JSON and produces a chunk of HTML. New block types drop in by extending `ContentBase`.

## Using colvmn as a submodule

```
git submodule add https://github.com/stevedekorte/colvmn.git colvmn
```

Then each page's `index.html` loads `/colvmn/style.css` and `/colvmn/layout/layout.js`, with an `_index.md` sibling describing the content. Run `node colvmn/static-gen.js` from the site root to regenerate static HTML, `sitemap.xml`, `llms.txt`, and `llms-full.txt`.

## Self-hosting

colvmn can render its own pages without being a submodule of itself — pass the site root explicitly:

```
node static-gen.js .
```

The root `index.html` references `./style.css` and `./layout/layout.js` directly.
